## Executive Summary

The site is visually polished and structurally sound, but it fails the bar for an international artist brand on several non-negotiables: **data integrity** (Brodyaga view counts and Spotify cover URLs are inconsistent across pages), **localization completeness** (Events never reads RU/EL titles; several user-facing strings are hardcoded English), **indexable SEO** (raw `<a>` anchors break locale routing; `youtube.com` embeds instead of `youtube-nocookie.com`), **security** (HTML/email injection in the Resend template, `'unsafe-eval'` in CSP), and **reliability** (in-memory rate limiting is a no-op on Vercel serverless; Analytics doesn't re-fire on App-Router navigations).

**Severity legend:** 🔴 Critical · 🟠 High · 🟡 Medium · 🟢 Low

## 1. UX / UI — Artist-Standard Experience

### 1.1 🔴 Duplicate cover art for two singles
`src/app/[locale]/music/page.tsx:225` and `:240` both point at `i.scdn.co/image/ab67616d00004851f57422e4fe7802226d1e613f`. Brodyaga and Ty Vsyo Poteryala render with the identical cover — a credibility-killer for a music site. Fix: pull each single's cover directly from `discography.ts` or use Spotify's oEmbed per `spotifyUri`.

### 1.2 🟠 Navbar vs. MobileMenu inconsistency
`src/components/layout/Navbar.tsx` omits `contact`, while `src/components/layout/MobileMenu.tsx` includes it. Desktop users have no direct nav link to the booking page. Fix: make both render from one shared array in `src/config/nav.ts` (create it).

### 1.3 🟠 Mobile menu is not accessible
`MobileMenu.tsx` has: no Escape-key close, no focus trap, no body-scroll lock, no `aria-modal`, no `role="dialog"`. Touch users can scroll the page behind the overlay; keyboard users can tab out of it.

### 1.4 🟠 Gallery lightbox — keyboard + focus management
`src/components/gallery/GalleryClient.tsx` uses `<div onClick>` on thumbnails (no `role="button"`, no `tabIndex`, no Enter/Space handlers), no body-scroll lock when open, no focus restore on close, hardcoded EN `aria-label`s.

### 1.5 🟠 Press page — dead Download icons
`src/app/[locale]/press/page.tsx` renders "Press Photos" cards with `<Download />` icons, but no `href`/`download` attribute: pressing them does nothing. Either wire to an actual zip/image URL or drop the icon.

### 1.6 🟡 Events tabs lack ARIA semantics
`src/components/events/EventsTabs.tsx` is not a proper `role="tablist"` / `role="tab"` / `aria-selected` structure. Screen readers announce a list of buttons, not a tab set.

### 1.7 🟡 Hardcoded `aria-label`s across components
`Navbar.tsx` ("Open menu"), `MobileMenu.tsx` ("Close menu"), `YouTubeFacade.tsx` ("Play {title} on YouTube"), `SinglesGrid.tsx` ("Play {title}"), `layout.tsx` ("Skip to content") — none are localized. RU/EL users hear English from the screen reader.

### 1.8 🟡 Links page hydration flash
`src/app/links/page.tsx` is client-only and does `useState('Greek Soul · Eastern Sound')` + `navigator.language` detection in `useEffect` — the translation swaps visibly after paint.

### 1.9 🟢 `prefers-reduced-motion` usage
ScrollReveal/HeroSection/PageHero correctly render visible SSR state and only attach Framer Motion below the fold on mount. Good.

## 2. Internationalisation (EN / RU / EL)

### 2.1 🔴 Events page ignores localized titles
`src/lib/data/events.ts` provides `titleRu`/`titleEl` for every event, but `src/app/[locale]/events/page.tsx` reads `event.title` directly. Every RU/EL visitor sees English event names. Same issue with `eventSchemas` — the JSON-LD published to Google is always English.

### 2.2 🟠 Hardcoded English strings in pages
- `src/app/[locale]/contact/page.tsx:50` — `"Booking Manager: Liana"` (no i18n key).
- `src/app/[locale]/contact/page.tsx:58` — `"Phone / WhatsApp:"`.
- `src/app/[locale]/videos/page.tsx:109` — `` `Filmed in ${video.shotLocation}` `` (English preposition hard-baked).
- `src/components/music/SinglesGrid.tsx:82` — `"plays"` label.
- `src/components/music/SinglesGrid.tsx:197` — `"No releases for this year."`.
- `src/components/BookingForm.tsx:73` — `"Something went wrong. Please try"` error fallback.
- `src/app/[locale]/music/page.tsx` — `10.4K` Spotify monthly listener count hardcoded; should be an i18n-formatted number.

### 2.3 🟠 WhatsApp locale routing incomplete
`src/lib/utm.ts` `getWhatsAppUrl(locale)` differentiates only RU; EL falls back to the EN message template. Greek visitors get an English pre-filled WhatsApp message.

### 2.4 🟠 Unused i18n keys
`messages/en.json` / `ru.json` / `el.json` all contain `music.meta_description`, but `src/app/[locale]/music/page.tsx` hardcodes `musicDescriptions` locally — the translation is never consumed.

### 2.5 🟡 `videos/page.tsx:14` viewsLabel map
```
const viewsLabel = { en: 'views', ru: 'просмотров', el: 'προβολές' }
```
Should live in `messages/*.json` under a `videos.views_label` key, not in page code.

### 2.6 🟢 Message parity
en/ru/el all 242 lines with matching structure. Cyrillic + Greek fonts loaded with proper `subsets: ['cyrillic']`/`['greek']` in `layout.tsx`. Native diacritics preserved.

## 3. SEO / Indexation

### 3.1 🔴 Brodyaga view-count inconsistency (trust signal)
`src/lib/data/videos.ts:45` = `views: "69M+"`; `:51` context says `"69M+ YouTube views"`; `src/app/links/page.tsx:9` says `"Watch Brodyaga (69M+ views)"`. Legacy copy elsewhere references `22M+`. Google indexes both figures → rich-results mismatch + brand inconsistency. Pick one, update all.

### 3.2 🔴 Raw `<a href>` breaks locale
- `src/app/[locale]/music/page.tsx:281` — `<a href="/videos">`
- `src/app/[locale]/videos/page.tsx:59,114` — `<a href="/contact">`
- `src/components/music/SinglesGrid.tsx:183` — `<a href="/contact">`

These bypass `next-intl`'s `Link` and drop RU/EL visitors on the default locale (or a 404 depending on middleware path). Fix: import `Link` from `@/i18n/routing`.

### 3.3 🟠 Wrong schema type for a solo artist
`src/lib/seo.ts` `generatePerformerSchema` uses `@type: PerformingGroup`. Alexandros is a solo artist — should be `MusicArtist` or `Person` (with `sameAs`/`memberOf` if relevant). Google's Knowledge Graph won't merge a PerformingGroup with a solo artist brand entity.

### 3.4 🟠 YouTube domain mismatch in CSP vs. embed
`next.config.mjs` `frame-src` allows `youtube.com` but NOT `youtube-nocookie.com`. If you switch `YouTubeFacade.tsx` to the privacy-enhanced domain (recommended; see 4.1), update CSP too.

### 3.5 🟡 `/links` correctly excluded from sitemap
`src/app/sitemap.ts` covers 3 locales × 8 canonical pages with reciprocal `hreflang` + `x-default`. `/links` is `robots: index:false` and intentionally absent. Good.

### 3.6 🟡 Booking confirmation mail — EN only
`src/app/api/booking/route.ts` sends the same confirmation body regardless of request locale. Include `locale` in the form payload and branch the template.

### 3.7 🟢 robots.txt / sitemap
`src/app/robots.ts` allow-all + sitemap reference is correct. `sitemap.ts` emits fully qualified URLs with `www` canonical. Good.

## 4. Media Pages (Spotify / YouTube)

### 4.1 🟠 YouTube embed is not privacy-enhanced
`src/components/YouTubeFacade.tsx` uses `https://www.youtube.com/embed/${videoId}`. Despite a CookieConsent banner, the embed drops YouTube cookies on first frame. Swap to `https://www.youtube-nocookie.com/embed/${videoId}` AND update CSP `frame-src` in `next.config.mjs`.

### 4.2 🟠 Thumbnail fallback works but `alt` is hardcoded EN
`YouTubeFacade.tsx` falls through `maxresdefault → sddefault → hqdefault → mqdefault` — that's good; but `aria-label='Play ${title} on YouTube'` is English-only.

### 4.3 🟠 LatestRelease streaming embed ignores selected single
`src/components/home/LatestRelease.tsx:87` — `StreamingEmbed` receives the default artist URI, not the URI of whichever single the user selected from the grid. Selecting "Ty Vsyo Poteryala" in the UI but the Spotify player keeps playing the default album. Also `t('latest_description')` is static — doesn't change per selected single.

### 4.4 🟡 Missing album artwork on disk
`src/lib/data/discography.ts` references `/images/albums/*.jpg` for 13 singles + 1 album. `public/images/albums/` does not exist. Site relies on `spotifyCoverUrl` fallback — fine until a Spotify image URL 404s, at which point the UI shows a broken image.

## 5. QA / Functional Correctness

### 5.1 🔴 HTML/email-template injection in Resend call
`src/app/api/booking/route.ts` (approx. lines 75–83 and 93–99) builds the HTML body with string interpolation of user-controlled fields (`name`, `email`, `message`, etc.). A visitor can inject markup, attribute-break, or smuggle an `<a href="javascript:…">` into your mail client. Fix: escape HTML entities for each field, or switch to Resend's React-email templating.

### 5.2 🔴 In-memory rate limit broken on serverless
`src/app/api/booking/route.ts` uses a module-scope `Map<ip, count>`. Every Vercel serverless invocation can spin up a fresh container — the Map is empty on 70%+ of requests. The rate limit provides false assurance. Fix: Upstash Redis, Vercel KV, or `@upstash/ratelimit`.

### 5.3 🟠 Booking form accessibility
`src/components/BookingForm.tsx` — labels lack `htmlFor` + matching `id` on the input. Submit label during send is `'...'` (no i18n key). Honeypot uses `opacity-0 -z-10 h-0` (good), but has no `tabindex={-1}` or `autocomplete="off"`.

### 5.4 🟠 No email regex on API side
`api/booking/route.ts` doesn't validate email format server-side; relies entirely on `react-hook-form` client validation, which is trivially bypassed.

### 5.5 🟠 CookieConsent — decline reloads, accept doesn't fire
`src/components/CookieConsent.tsx` reloads the page on decline (jarring) and only sets state on accept — so analytics don't actually fire until the next navigation. Combine with 5.6.

### 5.6 🟠 Analytics not wired to App Router SPA navigations
`src/components/Analytics.tsx` reads consent once on mount and never re-fires on route change. In an App Router SPA, client transitions don't reload the page — you're losing pageviews for any user who clicks through the site. Fix: `usePathname()` + `useSearchParams()` effect to push GA4 `page_view` events.

### 5.7 🟡 Press page "Download" is decorative only (see 1.5).

## 6. Code Quality & Engineering

### 6.1 🟠 `'unsafe-eval'` + `'unsafe-inline'` in CSP
`next.config.mjs` `script-src` allows both. `next/script` inline bootstraps require `unsafe-inline` without nonces, but `unsafe-eval` is unnecessary for this codebase. Drop `unsafe-eval`, add a nonce strategy for `unsafe-inline` if you can.

### 6.2 🟠 Error tracking is `console.log`
`src/lib/error-tracking.ts` is a shim — no Sentry/Rollbar integration. Silent failures in production.

### 6.3 🟡 `.env` committed to git
`/home/user/tsopozidis/alexandros-tsopozidis-website/.env` contains `NEXT_PUBLIC_GA4_ID=G-SKPD3HKR2Y` — public and safe to expose, but `.gitignore` only ignores `.env*.local`, not `.env`. Policy risk: a future contributor adding `RESEND_API_KEY` to the same file will leak it. Either rename to `.env.local` (already honored by Next) or extend `.gitignore` and leave a `.env.example` only.

### 6.4 🟡 No test framework
`package.json` has no Jest/Vitest/Playwright. For a content-light site this is defensible, but at minimum add a Playwright smoke suite hitting the 8 canonical routes × 3 locales to catch the anchor/locale bugs (§3.2) and Events localization (§2.1).

### 6.5 🟡 `tsconfig.json` loose
`noUnusedLocals: false`, `noUnusedParameters: false`. Flip to true or accept that dead imports accumulate.

### 6.6 🟢 TS strict + strictNullChecks enabled. Good.

### 6.7 🟢 `src/middleware.ts` canonical-host + skip-list + 307→301 root patch is correct and reasonable.

## 7. Vercel Deployment

### 7.1 🟠 Rate-limit storage must move off the serverless instance (see 5.2). This is a Vercel-specific failure mode.

### 7.2 🟡 `images/*` Cache-Control
`next.config.mjs` sets long Cache-Control on `/images/*`. Fine, but Vercel's default `immutable` only applies when the URL contains a content hash. Static files under `public/images/` don't get hashes — update logic or rely on Vercel's default.

### 7.3 🟡 Preview builds will break on missing env
Add a CI check that fails a preview deploy if `RESEND_API_KEY` is absent — currently the booking endpoint silently 500s.

### 7.4 🟢 Apex → www redirect correctly happens at middleware + `next.config.mjs`. No duplication flagged.
